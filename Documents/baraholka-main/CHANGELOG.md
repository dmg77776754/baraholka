### 5. ✅ Документация по безопасности (SECURITY.md)

**Поясняет:**

**1. Аутентификация через Telegram**
- Невозможно подделать личность пользователя
- Каждый пользователь имеет уникальный `telegram_user_id`
- Telegram WebApp гарантирует подлинность данных

**2. Row Level Security (RLS)**
- Все операции защищены политиками на уровне БД
- Примеры всех политик и как они работают
- Почему ANON ключ безопасен (работает только через RLS)

**3. Защита от распространенных атак**
- XSS (Cross-Site Scripting) — React автоматически экранирует
- SQL Injection — параметризованные запросы Supabase SDK
- CSRF — обработка через CORS и токены
- Rate Limiting — требуется для production

**4. Загрузка файлов**
- Проверка MIME type
- Ограничение размера (5MB)
- Компрессия изображений

**5. Требования для production**
- ✅ HTTPS везде
- ✅ Проверка подписи `initData` на сервере
- ✅ Rate limiting
- ✅ Логирование и мониторинг
- ✅ Резервные копии БД

**6. Раскрытие информации**
- Какие данные видит публика
- Где может быть утечка (номер телефона)
- Как это можно исправить (маскирование)

## 🧪 Как протестировать

### Быстрый тест (локально):

```bash
cd baraholka
npm install
npm run dev
# Открыть http://localhost:5173
# Эмулировать мобильный в DevTools (Ctrl+Shift+M)
```

### Тест всех функций:

1. **Публикация объявления:**
   - Нажать "➕ Подать объявление"
   - Заполнить форму
   - Загрузить фото
   - Опубликовать

2. **Просмотр объявления:**
   - Вернуться на главную
   - Кликнуть на объявление
   - Посмотреть фото-галерею
   - Посмотреть контакты

3. **Управление своими:**
   - Нажать "📋 Мои объявления" (шапка)
   - Выбрать объявление
   - Кликнуть на него
   - Удалить или посмотреть детали

4. **Администраторская функция:**
   - Нажать "⚙️" (администратору)
   - Одобрить/удалить объявление
   - Проверить, что оно исчезло из "На модерации"
